## Objective

`list` should be updated so that it:

- List from current user by default (allow all users via flag)
- Order by `submit` time
- Colorize!

## Context

The `chain list` command previously showed all workflows across all users, with no sorting and no visual formatting. This made it hard to quickly scan your own recent jobs.

## Summary

All changes land in `src/seekr_chain/cli.py` only (backend unchanged).

**Default user filter** ‚Äî added `-A/--all-users` flag; when neither flag is given, `effective_user` falls back to `os.environ.get("USER")`, matching how submit labels workflows. `--user` still overrides explicitly; `--all-users` wins over `--user` if both are passed.

**Sort by submit time** ‚Äî `workflows.sort(key=lambda wf: wf["created"])` after fetch. `creationTimestamp` is ISO 8601 (`2026-01-01T00:00:00Z`), which sorts correctly as a plain string. Sort is client-side because Kubernetes list API doesn't support server-side ordering. If `--limit` is set it pages from k8s first, then sorts that page ‚Äî fine for typical usage.

**Colorize** ‚Äî `rich.text.Text` with `style=` per cell. Color map: `green`=Succeeded, `cyan`=Running, `yellow`=Pending, `red`=Failed/Error/Terminated, `dim`=Omitted/Skipped. Argo uses capitalized phase strings (e.g. `"Succeeded"`, not `"SUCCEEDED"`).

**Timestamp formatting** ‚Äî `_format_timestamp()` parses the UTC ISO string and formats as `"Mon DD HH:MM"` in the local system timezone using `datetime.astimezone()`. The raw ISO string is still used internally for sorting before display.

**Tests** (`tests/unit/test_cli.py`): `test_default` updated to use `patch.dict(os.environ, {"USER": "testuser"})` for hermeticity. Added `test_all_users`, `test_explicit_user_overrides_default`, `test_all_users_overrides_user_flag`, `test_sorted_by_created`.

**Gotcha** ‚Äî the existing `TestListWorkflows` tests patch `seekr_chain.backends.argo.list_workflows.kubernetes` at module level. Moving `import kubernetes` inside the function body (to defer the slow import) breaks this pattern. Reverted; the module-level import stays.

**Parallelism note** ‚Äî user asked about parallelizing because `chain list` takes ~10s. The implementation makes a single `list_namespaced_custom_object` call; there is nothing to fan out. The latency is likely auth token refresh (exec credential provider) + network round-trip to the API server. Multi-namespace parallel fetch would be the only real opportunity if that use case arises.
